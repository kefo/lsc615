Part 1 - Descriptive Metadata (10% of overall grade)

Due (Updated, 1 October 2012, new date and time): Post XML records to this week's discussion forum by 4 October, 11:59 PM; Package of XML records and essay due by 11 October, 11:59 PM.


Create two CDWA-lite records (one for the audio recording and one for the score) for the folksong. For the CDWA-lite records use the XML schema found at http://www.getty.edu/CDWA/CDWALite/CDWALite-xsd-public-v1-1.xsd. The CDWA-Lite schema and elements dictionary available here: http://www.getty.edu/research/publications/electronic_publications/cdwa/cdwalite.pdf. Examples of CDWA records may be found here: http://3windmills.com/lsc615/resources/cdwalite/.


You may feel free to look at descriptive records for the two expressions while creating your own. A MODS record for the MP3 sound recording can be found at http://memory.loc.gov/diglib/ihas/loc.afc.afcss39.2633a2/mods.xml, and a MODS record for the musical score can be found at http://lcweb2.loc.gov/diglib/ihas/loc.music.sm1847.420580/mods.xml.


You will also create a technical metadata record for the audio file using the AudioMD (http://www.loc.gov/standards/amdvmd/audioMD.xsd) and a technical record for one of the TIFF files using the MIX Schema (http://www.loc.gov/standards/mix/mix20/mix20.xsd). Examples of AUDIOMD records are available at http://3windmills.com/lsc615/resources/audiomd/ and examples of MIX records are available at http://3windmills.com/lsc615/resources/mix/ (the second example is pretty much a representation of all the possible values, but bear in mind that it is for MIX version 1.0, not 2.0; I believe 2.0 is backwards compatible). These standards require you to record technical characteristics about the data file. I recommend saving the files to your local hard drive and then loading them up in one or more display/playback tools you may have and seeing what technical information the tools can provide. iTunes "Get Info" option, for example, can tell you an audio file's format, size, bit rate, sampling rate, number of channels, etc. Windows Media Player can also provide some technical information about the file. The same functionality is likely built in to an image or photograph organizing program for the TIFFs.


Then write a 1000-1100 word paper discussing any problematic issues that arose in creating the metadata records. Evaluate CDWA-lite's effectiveness in describing the resources. You should compare this experience with your experience creating DC records, MODS records, and VRA records. Also, explore how the technical metadata you created meshes with the descriptive metadata. How is it different? Consider some of the uses of technical metadata to a variety of users.


You should submit both your paper and complete copies of your XML metadata records. The paper should be submitted in MS-Word, OpenOffice or PDF format; your XML records should be submitted as four separate XML files. Please submit all of your files through Blackboard by packaging them together using WinZip, StuffIt or Unix tar+gzip and submitting the single, packaged file through Blackboard.

Part 3 - Crosswalking (10% of overall grade)

Due: (Updated, 1 October 2012, new time)1 November, 11:59 PM.


For this assignment, you will start with your CDWA-lite records you created in part 1 of this assignment. You'll crosswalk the record into simple Dublin Core format, using the OAI Dublin Core XML schema as the basis for authoring the new metadata record. Then, using only the information contained within the Dublin Core Record, crosswalk that record into IEEE LOM Metadata using the IMS Global Learning Consortium LOM Metadata schema (NB: there are slight variations between IEEE LOM and the IMS implementation of LOM. See the IMS Metadata Best Practice Guide for details on the IMS implementation). Write a 3 page paper (1-inch margins, double-spaced) discussing the changes in metadata resulting from each crosswalk, and the overall changes in metadata resulting from the complete process. Try to focus your discussion on why you choose to retain, eliminate or add information in each crosswalk, and how you think that may impact people's ability to use the resulting metadata record. Bundle your XML records with your paper and submit them through Blackboard.
